# Visual rendering matrix — committed baseline

Per-viewer subdirectories (`blenderbim/`, `bimvision/`, `fzk/`, `web-ifc/`)
hold one PNG per (fixture, angle) pair. The aggregate manifest lives at
`MANIFEST.json` and records every PNG's SHA-256 hash.

**Layout once seeded:**

```
tests/baselines/visual-matrix/
  MANIFEST.json
  blenderbim/
    simple_box_iso-ne.png
    simple_box_iso-nw.png
    simple_box_iso-se.png
    simple_box_iso-sw.png
    multistorey_residential_iso-ne.png
    …
  bimvision/
    …
  fzk/
    …
  web-ifc/
    …
```

Total target: 3 fixtures × 4 viewers × 4 angles = **48 PNGs**.

**Phase 0 status:** deferred. The directory exists so the workflow YAMLs
have something to point at; the screenshots are captured in a separate
session because:

- BlenderBIM CLI requires `blender` + the Bonsai add-on installed (works
  in CI with the apt `blender` package).
- BIMVision has **no CLI** — these tiles must be captured manually
  by a human running the BIMVision GUI on Windows. See
  `BIMVISION_MANUAL_CAPTURE.md` (auto-generated by the runner).
- FZK Viewer's CLI is Windows-only — capture happens on a Windows runner
  in the release workflow.
- web-ifc capture relies on a Puppeteer driver
  (`scripts/puppeteer/visual-matrix.js`) which has not yet been authored.

## How to seed this directory

```bash
cd neobim-ifc-service
python scripts/capture_visual_matrix.py \
  --fixtures simple_box,multistorey_residential,non_rectangular \
  --viewers blenderbim,fzk,web-ifc \
  --rich-mode full
# Manually capture BIMVision tiles per BIMVISION_MANUAL_CAPTURE.md.
```
